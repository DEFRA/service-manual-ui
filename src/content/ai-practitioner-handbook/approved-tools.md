Use only AI tools from the Defra Tools Guidance. This ensures your work meets government security standards and data protection requirements.

## Why this matters

Approved tools have been assessed for security, data handling, and compliance with government policy. Using unapproved tools risks exposing sensitive data and breaching organisational standards. See [clear governance is essential](/ai-practitioner-handbook/ai-governance) for lessons learned on this topic.

## How to request a new tool

If you need an AI tool that is not currently approved:

1. Contact the AI Capabilities and Enablement team at [cctsassurance@defra.gov.uk](mailto:cctsassurance@defra.gov.uk).
2. Explain what the tool does and why you need it.
3. Describe what data you plan to use with it.
4. Wait for the team to complete their assessment before using the tool.

The team will evaluate the tool against security, data protection, and suitability criteria. This process exists to protect you and the organisation, not to block progress.

## MCP servers

Model Context Protocol (MCP) servers are currently under review. Their use is limited to pilot projects with explicit approval from the AI Capabilities and Enablement team. Do not deploy MCP servers in production without authorisation.

## Further reading

See the [Defra AI SDLC Tool Guidance](https://github.com/DEFRA/defra-ai-sdlc) for the full list of approved tools and detailed usage guidance.
