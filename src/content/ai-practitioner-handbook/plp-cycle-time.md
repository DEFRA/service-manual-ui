The PLP delivery team introduced GitHub Copilot between September and November 2025. They measured the impact on their development workflow over that period.

## Results

- **61% reduction in cycle time** — tasks moved from backlog to done significantly faster
- **50% of new features** included AI-assisted documentation
- **Faster code reviews** — Copilot-generated code was more consistent and easier to review

## How AI helped

The team used GitHub Copilot for 4 main activities:

- **Code generation for boilerplate** — repetitive code patterns were generated quickly, freeing developers to focus on business logic
- **Inline documentation** — Copilot suggested comments and docstrings as developers wrote code
- **Pipeline automation** — CI/CD configuration and deployment scripts were drafted faster
- **Architecture documentation** — technical design documents were produced with AI assistance

## Challenges

**Initial resistance.** Some team members were sceptical about AI-generated code quality. Demonstrating measurable results helped build confidence.

**Competing priorities deprioritised learning.** Sprint commitments left little time for developers to learn Copilot properly. Dedicated learning time was essential.

**Inconsistent premium access.** Not all team members had Copilot licences at the same time. This created an uneven experience across the team.

**Training alone was not enough.** One-off training sessions did not change behaviour. The team needed ongoing support to build habits.

## What worked

**Regular check-ins** kept AI adoption on the agenda. Weekly discussions helped the team share tips and solve problems together.

**Visible champions** within the team showed others what was possible. Seeing a colleague use Copilot effectively was more persuasive than formal training.

**Measurable impact data** built organisational support. The 61% cycle time reduction gave leadership confidence to invest further in AI tooling.
